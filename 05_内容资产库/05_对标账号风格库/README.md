# 对标账号风格库

这里保存已经逐账号拆解完成、可以在写稿时自动调用的文案风格资产。

## 一账号一档案

每个账号必须保持独立目录，至少包含：

- 机器可读的风格指纹；
- 独立的语言风格指纹和开头表达指纹；
- 账号文案说明书；
- 全量逐视频证据索引；
- `07_对标表层素材库.jsonl`，完整保存来源文本及其例子、案例、比喻、个人故事、具体事实、数字和顺序，供分析、溯源与核验；
- 可复用的结构、钩子、节奏、论证、情绪路线模型及风险边界；
- 原始转写项目的路径。

未完成单账号拆解前，不把它与其他账号混合总结。只有账号级指纹稳定后，才登记到 `registry.json` 并允许认知类内容 skill 自动调用。

## 写稿时的组合规则

1. 一篇稿只选择一个主账号、一个结构模型和一个开头家族控制整体骨架。
2. 其他账号最多贡献两项局部能力，例如钩子、故事论证或诗性结尾。
3. 优先匹配内容要完成的情绪任务，再匹配主题关键词。
4. 每次写稿都从主账号的来源档案检索相关记录；迁移其中的钩子功能、结构、节奏、论证和情绪路线，并以新的原创建议、场景、例子、比喻、故事和措辞完成稿件。
5. 每篇稿保留分析素材ID、段落ID、来源、转化说明和事实核验状态；具名事实、数字、研究、引用和当前信息发布前核验。
6. 发布后把数据回收到对应账号、结构模型、开头家族和素材ID。

## 状态门槛

- `采集中`：只有链接或少量原文，不进入注册表。
- `拆解中`：已建立转写和逐视频证据，不自动写稿调用。
- `可调用`：已形成账号说明书、统一风格指纹、语言指纹、开头表达指纹、结构模型、完整来源档案和风险边界，可登记到注册表。
- `已验证`：至少有用户自己的三篇稿采用该风格发布，并带回留存、完播、收藏、转发和关注数据。
