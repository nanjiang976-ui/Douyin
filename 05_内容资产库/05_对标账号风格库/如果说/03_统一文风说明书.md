# “如果说”统一文风说明书

> 分析来源：65条人工终审稿。调用范围：统一账号文风与多个结构模型；不使用账号阶段。

## 核心表达机制

这个账号最稳定的能力，是把观众说不清的焦虑、关系困扰或行动困难，改写成一条可理解、可代入、能获得情绪价值的认知路径：

> 命名困境 → 打破默认解释 → 放进生活场景 → 给出新的理解 → 情绪修复或行动出口。

## 适配主题与观众感受

- 主题：认知提升、个人成长、自我提升、长期主义、关系、情绪、人生选择、传统智慧、人物故事、赚钱意识和行动力。
- 典型观众：焦虑、内耗、方向不清，但希望获得清醒感与行动感的成年人。
- 最终感受：被说中、被理解、突然想通、重新获得边界或掌控感。

## 钩子与前30秒

1. 先给一个反常识判断、具体困境、身份对照或概念误读。
2. 立刻让观众看到自己的生活镜像，而不是自我介绍。
3. 明确本条只解决一个问题：看清、停止内耗、建立判断或迈出下一步。
4. 在前30秒内完成一次具体的重新定义：把观众以为的原因，换成更能解释代价的原因；不得套用固定反转句式。

### 开头固定表达的调用方式

开头不只记录“反常识、提问、身份对照”等类别，还要单独选择一个开头家族，并明确前三拍：**截停 → 观众照见 → 欠答案/收益**。

本账号可调用的七个家族是：熟悉判断翻面、痛点机制重命名、熟语缺口、同场景判断分层、具体账本与隐性代价、负面状态反向解释、无反馈生存题。具体句式骨架、情绪按钮、承接动作和禁用边界见 `05_开头表达指纹.json`。

固定的是功能骨架，不是原句。最近三条脚本不得使用完全相同的第一拍；每次都要更换主题、场景、变量、证据和表层措辞。开头提出的机制、收益或问题必须在正文第一部分兑现。

## 语言与节奏

- 开头短、判断先行；中段在概念、场景、故事和解释之间切换；短句只在证据或因果之后落点；结尾放慢。
- 高频功能：重新定义、诊断式反问、自我镜像、因果展开、边界补充和二次转折。
- 通常 1—2 句一段，一段只完成一个推进动作；每段应给理解、场景、机制或边界中的一种回报，避免连续抽象讲道理。
- 结尾不增加新概念，优先回到开头困境，交付清醒、边界、一个自问或一个低门槛动作；不做动员排比。
- 具体的语气、段落呼吸、转折和排比边界，以 `04_语言风格指纹.json` 为准。

## 调用边界

- 可以迁移：选题重构、钩子功能、论证顺序、句子长短交替、情绪路线和结尾功能。
- 可直接调用：原句、个人故事、标志性比喻、人物案例顺序、账号口头禅。
- 写稿只选择一个主模型；如确有必要，再从其他已登记账号调用不冲突的局部能力，最多两项。
