# 内容资产库使用说明

这个文件夹用于沉淀你从对标账号高赞视频中提炼出来的长期可复用资产。

`02_账号项目` 用来保存某个具体账号的原始材料、转写文本和拆解报告。

`05_内容资产库` 用来保存你自己的通用资产：选题、钩子、脚本结构。这里不直接照搬对标账号内容，只保存经过转译后的创作逻辑和模板。

当前账号的固定定位见 `00_认知类账号定位.md`：主线是认知、个人成长和自我提升。AI不是默认方向，只有明确服务于认知主题时才作为辅助场景出现。

`05_对标账号风格库` 保存每个账号独立的机器可读风格指纹，并由 `registry.json` 统一登记。认知类内容 skill 写稿前会自动读取这个注册表，按主题选择一个主账号、一个结构模型和最多两个辅助能力，不需要每次重新提醒。

后续由 SRT 开始学习对标文案时，使用 `douyin-benchmark-script-learning`：SRT 直接转为纯文字文案，再把每个账号沉淀为一个统一风格指纹、一个语言指纹、一个开头表达指纹和多个结构模型。写稿时选择“主账号 + 主结构模型 + 开头家族”，不再区分账号阶段。

`06_研究资料与案例母库` 保存从对标文案中发现、但仍需独立查证的研究母题、案例方向和事实风险。这里的条目是研究队列，不是可以直接引用的事实库。

## 推荐工作流

1. 把想拆解的账号名称和视频链接填入 `04_待拆解清单/待拆解清单.csv`。
2. 每个对标账号的原始分析结果放到 `02_账号项目/账号名/03_分析产出/`。
3. 拆解完成后，把可复用内容分别归档到：
   - `01_选题库/选题库.csv`
   - `01_选题库/如果说_终审选题资产.csv`
   - `02_钩子库/钩子库.csv`
   - `02_钩子库/如果说_终审钩子资产.csv`
   - `02_钩子库/高留存开头表达手册.md`
   - `03_脚本结构库/脚本结构库.csv`
   - `03_脚本结构库/如果说_终审结构资产.csv`
   - `04_表达与论证模型库/如果说_终审表达论证模型.csv`
   - `05_对标账号风格库/账号名/风格指纹.json`
   - `05_对标账号风格库/账号名/开头表达指纹.json`
   - `06_研究资料与案例母库/账号名_研究核验队列.csv`
4. 在 `05_对标账号风格库/registry.json` 登记已经完成拆解、允许写稿时自动调用的账号。
5. 每拆 10 条视频，至少反向生成 5 个你自己的选题，避免只研究不发布。

## 每次发给 Codex 的格式

```text
账号名称：
账号主页链接：
视频链接：
我想重点学习：选题 / 前3秒开头 / 情绪价值 / 钩子 / 脚本结构 / 观点表达 / 画面形式
优先级：高 / 中 / 低
备注：
```

## 入库原则

- 学逻辑，不抄内容。
- 原视频里的具体表达只用于理解，不作为你的成稿直接复用。
- 入库内容要转译成你当前认知类账号的方向：赚钱认知、长期主义、自我改变、行动力、普通人成长和人生选择。
- 对投资、医疗、法律等高风险观点，只标记为“创作者观点”，不要直接当事实使用。
- 写稿时只允许一个对标账号控制主结构；其他账号只能提供明确命名的局部能力，避免混成没有辨识度的平均腔调。
- 每次正式稿保留简短的对标能力调用记录，便于发布后把数据反向归因到具体风格和结构模型。
- 每次正式稿记录开头家族ID，并把3秒留存、5秒留存和完播率与点赞、收藏分开回收；没有真实留存数据时，不用互动量代替留存结论。
- 当前已完成终审的对标资产，以文件名含“终审”的版本为写稿优先源；早期资产保留为历史记录，不作为默认检索层。
