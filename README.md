# Douyin：公开内容采集、转写与创作分析工作台

这是一个面向抖音**公开视频**的个人研究与内容创作工作台。它把内容研究中的采集、转写、整理、拆解与资产沉淀放在同一套可复用的目录中，帮助创作者从优秀案例中提炼选题、钩子、论证结构和表达方法，并形成自己的内容策略。

> 目标是学习与分析，不是搬运或复刻。仓库只保留脚本、文本、分析结果和内容资产；原始视频、浏览器登录态与本地运行缓存不会上传。

## 能做什么

- 从抖音公开页面采集作品信息、页面文本和接口响应。
- 下载获授权或公开可访问的视频，使用 `faster-whisper` 进行本地离线转写。
- 按账号整理转写稿、截图、抓取数据和逐视频分析结果。
- 将账号研究沉淀为选题库、钩子库、脚本结构库、表达与论证模型库。
- 从已审校的研究数据生成内容表格、账号报告和可复用的创作参考资料。

## 目录结构

```text
01_工具_采集转写分析/       可复用的采集、转写、整理与分析脚本
02_账号项目/                按账号管理的研究项目、转写文本和分析产出
03_浏览器运行环境_勿上传/   本地 Edge 配置、缓存与登录态（已忽略）
04_运行缓存与经验记录/      截图、运行记录与可复用的流程经验
05_内容资产库/              选题、钩子、脚本结构和风格资产
06_创作测试/                自己的内容创作与表达测试
```

建议为每个研究账号使用一致的项目结构：

```text
02_账号项目/
  账号名称/
    01_原始视频/            仅保留在本地，不提交
    02_转写文本/
    03_分析产出/
    04_截图与视觉检查/
    05_抓取原始数据/
```

## 主要工具

| 工具 | 用途 |
| --- | --- |
| `douyin_profile_extract.js` | 连接本地 Edge 调试端口，提取公开主页或视频页的信息。 |
| `collect_public_transcripts.py` | 下载候选公开视频，并以 `faster-whisper` 进行离线转写。 |
| `transcribe_extracted_video.py` | 从单个抓取结果中提取视频信息并转写。 |
| `download_visual_checks.py` | 下载少量封面或图文素材，供人工核对。 |
| `build_*.py` | 将审校后的逐视频数据汇总为转写索引、内容表和分析报告。 |

## 使用方式

这是一个持续演进的研究工作台，不是开箱即用的通用产品。开始使用前：

1. 根据目标账号创建项目目录，并在脚本中确认账号 ID、候选视频和输出路径。
2. 如需转写，准备 Python 环境并安装 `faster-whisper`；部分采集脚本还需要 Node.js 与本地 Edge 调试环境。
3. 先处理少量公开视频，人工校对转写和分析结果后，再把可复用结论写入 `05_内容资产库/`。
4. 将新产生的原始视频、浏览器数据和运行缓存保留在本地，不要提交到 GitHub。

现有脚本中保留了一些历史研究任务的固定参数。复用前请先检查并替换账号、视频范围和输出位置，避免不同账号的数据混在一起。

## 使用边界

- 仅处理公开可访问的内容，并遵守平台规则、版权与相关法律。
- 不绕过私密权限，不采集或传播账号登录态、Cookie 或浏览器配置。
- 不批量搬运原始视频；研究结论应转化为自己的选题和表达，而非直接复制。
- 发布内容前，仍应自行核验事实、引用和版权风险。

## 项目定位

本仓库服务于一个长期内容系统：用公开案例研究建立可复用的创作方法，再将方法沉淀为选题、脚本和表达资产，最终支持稳定的内容生产与复盘。
