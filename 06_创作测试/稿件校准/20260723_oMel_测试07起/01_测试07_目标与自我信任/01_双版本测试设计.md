# 测试07｜双版本测试设计

## 1. 测试目的

同一选题分别制作3—5分钟版与6—7分钟版，检验oMel风格在不同内容承载量下是否都能保持：

- 开头快速进入具体动作；
- 人格自责转为可解释机制；
- 证据和生活场景及时兑现；
- 最终只落到一个当天能够完成的动作。

## 2. 两版共同变量

- 选题：为什么目标定得越大，你反而越不相信自己？
- 核心观点：反复把宏大愿望写成承诺又失约，会削弱下一次自我承诺的分量；重建自我信任需要一条当天能够兑现的行动证据。
- 主对标：oMel。
- 优先主模型：`omel.model.attention-mechanism.v1`。
- 主情绪路线：羞耻/自我怀疑 → 被准确看见 → 听懂反馈机制 → 重新获得掌控感。
- 核心思想母题：研究后从同一组候选中确定，原则上两版共用。
- 开头：原则上两版共用同一个用户钩子组合；若因时长必须改变，标记为非严格时长对照。
- 核心证据：至少保留同一条最强研究证据和同一个核心复合场景。
- 结尾：同一个“当天兑现一次”的行动，不分别创造两套方法。

## 3. 差异变量

| 维度 | 3—5分钟版 | 6—7分钟版 |
|---|---|---|
| 目标 | 单论点高密度表达 | 完整解释反馈链与适用边界 |
| 结构 | 2—3个推进阶段 | 3—4个清晰章节 |
| 场景 | 1个核心复合场景 | 同一核心场景回返，可增加1个不同领域验证 |
| 证据 | 1—2条最强证据 | 2—3条互不重复的证据 |
| 机制 | 解释“失约如何改变下一次行动预期” | 进一步区分目标难度、反馈速度、自我效能与合理失败 |
| 反方 | 1个简短边界 | 完整处理“小目标是否降低上限”和“失败是否必然伤害自信” |
| 行动 | 1个24小时内动作 | 同一动作，加判断标准和复盘方式 |
| 风险 | 容易过度简化 | 容易重复、灌水或把方法写成清单 |

## 4. 场景与真实性规则

- 用户允许灵活创作故事细节。
- 所有未由用户确认的经历均标记为`composite_scene`，只承担观众照见或机制演示功能。
- 不把虚构经历写成用户真实履历、真实收入、真实失败或真实结果。
- 如果需要第一人称增强个人IP，先提供候选故事给用户认领和修正，再进入正式口播。

## 5. 对比有效性

- 两版保持题目、核心主张、主对标、核心母题、核心证据和结尾动作不变，才可以作为严格的时长承载力对照。
- 若研究导致两版采用不同证据主轴、开头或结论，应标记为`非严格同题双版本`，不宣称差异仅由时长造成。
- 内部分数只判断文本质量与对标机制，不推断真实留存、完播或互动效果。
