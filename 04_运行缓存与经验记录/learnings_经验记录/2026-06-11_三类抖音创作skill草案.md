## 0. 总体设计

这套能力拆成 3 个 skill：

1. `douyin-cognition-script-builder`：认知类账号的选题、资料搜集、论据扩充、文案创作。
2. `douyin-ai-solopreneur-script-builder`：AI 知识、一人公司、自由职业、小而美生意方向的资料搜集、教学脚本和商业承接文案。
3. `douyin-editing-director`：把任意成稿文案拆成分镜，输出画面、AI 视频提示词、素材搜索词、字幕关键词和剪映执行建议。

三者关系：

```text
内容 skill
输入：话题 / 粗大纲 / 账号方向 / 目标观众
输出：研究资料 + 论据库 + 深度文案 + 视觉锚点

剪辑 skill
输入：定稿或半定稿文案
输出：分镜表 + AI 视频提示词 + 素材搜索关键词 + 字幕关键词 + 剪映执行方案
```

核心原则：

- 内容优先，剪辑服务内容。
- 文案必须有 2-5 秒钩子、总分总结构、5-10 秒刺激点、金句、真人感、利他性。
- 资料搜集必须区分事实、推断、观点，涉及外部事实、AI 工具、平台规则、价格、政策、公司动态时优先联网确认。
- 分镜不是机械一句话一个镜头，而是按“意义单元”拆解，再从每句话提取可视化关键词。
- 每个分镜至少包含：旁白、画面内容、AI 视频提示词、素材库搜索关键词、字幕关键词。

## 1. Skill 草案：douyin-cognition-script-builder

### 1.1 Frontmatter 草案

```yaml
---
name: douyin-cognition-script-builder
description: Build deep Douyin cognition/self-growth scripts from a topic, rough idea, social issue, human-nature question, or personal reflection. Use when the user wants 抖音认知类选题, 观点扩充, 深度文案, 论据搜集, 案例搜集, 总分总口播稿, 人味化脚本, 金句, 情绪点, 普通人改变, 赚钱意识, 长期主义, 自我成长, AI 时代普通人认知, or wants to turn a raw topic into a high-retention short-form video script with hooks, evidence, examples, and altruistic conclusions.
---
```

### 1.2 核心定位

把一个原始话题变成“有深度、有证据、有情绪、有真人感、有利他性”的认知类短视频脚本。

它不是只润色文字，而是承担：

- 研究员：搜集社会现象、案例、数据、理论、反方观点。
- 编剧：搭建钩子、总分总结构、情绪刺激点和金句。
- 论证助手：用案例和论据证明中心观点。
- 个人 IP 助手：把用户的经历、困境、价值观自然融入脚本。

### 1.3 适用内容方向

- 赚钱意识、长期主义、自我改变、普通人破局。
- 人性问题、社会焦点、阶层流动、内卷、焦虑、选择。
- AI 时代普通人能力变化，但重点仍是认知和个人成长。
- “为什么努力没有回报”“为什么普通人越来越焦虑”“为什么自由职业不是逃避上班”等。

### 1.4 工作流

#### Step 1：明确选题与观众

先判断：

- 目标观众是谁：普通上班族、学生、自由职业者、想学 AI 的人、想赚钱但迷茫的人。
- 观众为什么会停下来看：收益、冲突、恐惧、好奇、身份认同至少命中一个。
- 这条视频在内容漏斗中的位置：拉新、共鸣、信任、教学、转化。

输出：

```text
选题一句话：
核心观众：
观众当前痛点：
看完后获得：
视频漏斗位置：
```

#### Step 2：确定核心论点

每条视频只能有一个中心论点。先写清楚：

- 表层话题是什么。
- 真正要讲的底层机制是什么。
- 观众原来可能怎么想。
- 视频要把观众带到哪个新认知。

优先使用这几类解释框架：

- 激励结构：为什么人会被系统推着走。
- 成本/收益：为什么看似荒谬的选择其实有成本逻辑。
- 反馈机制：为什么人坚持不下去。
- 稀缺/边际价值：为什么有用不等于贵，努力不等于值钱。
- 集体困境：为什么每个人理性，整体却更累。
- 认知偏差：为什么人会误判机会、风险和自己。
- 时代迁移：旧规则为什么失效，新规则是什么。

#### Step 3：全面搜集资料和论据

如果需要外部事实，必须联网确认。搜集时至少覆盖：

- 社会现象：真实新闻、趋势、平台讨论、热点事件。
- 案例故事：普通人案例、公司案例、历史案例、名人故事。
- 数据证据：调查、报告、统计、行业数据。
- 理论框架：心理学、经济学、商业、社会学、传播学。
- 反方观点：为什么有人不同意，如何回应。
- 生活场景：观众每天能感受到的具体画面。
- 类比隐喻：让抽象观点更容易理解的比喻。

资料必须标注类型：

| 类型 | 说明 |
|---|---|
| 事实 | 有来源、可验证 |
| 推断 | 基于事实得出的判断 |
| 观点 | 适合口播表达的主张 |
| 案例 | 用来证明观点的人/事/场景 |
| 反例 | 防止文案片面 |
| 风险 | 避免过度承诺或误导 |

#### Step 4：生成论据库

输出一个可供脚本选择的素材池：

```text
核心案例：
数据/事实：
故事素材：
生活场景：
反方观点：
可用类比：
可用金句：
可视化锚点：
不建议使用的论据：
```

#### Step 5：搭建文案结构

默认总分总：

```text
开头 2-5 秒：钩子
总：提出问题/观点
分 1：第一个论点 + 案例/数据
分 2：第二个论点 + 案例/数据
分 3：第三个论点 + 案例/数据
总：结论/意义/行动建议
```

可选开头钩子：

- 反常识问题：为什么 X 明明这样，却会那样？
- 强对比痛点：同一个人做 A 可以很久，做 B 却坚持不了 5 分钟。
- 热点争议：最近很多人都在吵 X，但他们可能都忽略了 Y。
- 结果承诺：看完这条视频，你会重新理解 X。
- 真实困境：我也曾经遇到 X，所以我知道普通人最卡在哪里。
- 身份定位：我不是来讲大道理的，我是想用普通人能听懂的话讲清楚 X。

#### Step 6：写成短视频文案

硬性要求：

- 2-5 秒开头必须可单独成立，不能铺垫太久。
- 正文每 5-10 秒有一个刺激点：数据、冲突、金句、案例、画面锚点、反常识。
- 至少 2-3 个金句。
- 至少 1 个真人感元素：亲身经历、真实困境、学习过程、失败复盘、真实数据。
- 至少 1 个利他出口：观众看完知道下一步怎么做。
- 避免纯鸡汤，必须有具体案例、论据或解决办法。
- 结尾必须回答：这件事对普通人有什么意义。

#### Step 7：为剪辑 skill 输出视觉锚点

脚本最后附：

```text
需要可视化的数据：
适合用新闻/截图的位置：
适合用人物故事的位置：
适合用隐喻画面的位置：
适合大字压屏的金句：
适合停顿/重音的位置：
```

## 3. Skill 草案：douyin-editing-director

### 3.1 Frontmatter 草案

```yaml
---
name: douyin-editing-director
description: Turn a Douyin script into a detailed storyboard and editing plan. Use when the user has a script, outline, voiceover, or draft and wants 分镜拆解, 画面匹配, 可视化关键词, AI视频提示词, 素材库搜索关键词, 字幕关键词, 剪映操作, 转场, 特效, 音效, BGM, 字幕样式, 发布标题/标签, or full-process editing guidance for non-face or partial-face cognition, AI knowledge, tutorial, self-media, and one-person-company videos.
---
```

### 3.2 核心定位

把一篇文案变成新手也能执行的剪辑导演表。

它最重要的能力：

- 按合理逻辑拆分文案为“意义单元”。
- 从每句话/每段中提取“可视化关键词”。
- 为每个镜头设计画面、AI 视频提示词、素材库搜索关键词、字幕关键词。
- 告诉用户用剪映怎么做，而不是只说“加转场”“放 B-roll”。

### 3.3 输入要求

用户可以提供：

- 完整文案。
- 粗脚本。
- 内容 skill 输出的脚本和视觉锚点。
- 目标时长、账号方向、是否露脸、是否用 AI 合成声音、剪辑工具。

默认：

- 平台：抖音。
- 比例：9:16。
- 工具：剪映。
- 声音：AI 合成用户自己的声音。
- 风格：认知类/AI 知识类，不露脸或局部出镜。

### 3.4 工作流

#### Step 1：脚本诊断

先检查：

- 开头 2-5 秒是否抓人。
- 是否总分总。
- 每 5-10 秒是否有刺激点。
- 是否有金句。
- 是否有真人感。
- 是否有利他出口。
- 是否有需要补充视觉证据的地方。

如果文案本身不适合剪，先指出问题并给轻量修改建议。

#### Step 2：按意义单元拆分

不要机械一句话一个镜头。先按以下功能拆：

| 意义单元 | 典型时长 | 作用 |
|---|---:|---|
| 钩子 | 2-5s | 停住用户 |
| 问题提出 | 5-10s | 建立冲突 |
| 案例/故事 | 8-20s | 增加真实感 |
| 概念解释 | 5-15s | 提供深度 |
| 数据/证据 | 3-8s | 增强可信度 |
| 金句 | 1-3s | 制造记忆点 |
| 转折 | 2-5s | 推动继续看 |
| 结尾/CTA | 5-12s | 收束和行动 |

#### Step 3：提取可视化关键词

每个意义单元从文案中提取：

- 具体名词：电脑、办公室、地铁、账单、AI 工具。
- 行为动作：加班、搜索、写文案、做表格、剪辑。
- 情绪词：焦虑、兴奋、迷茫、被困住、松了一口气。
- 抽象概念：复利、反馈、边际成本、个人 IP、自由职业。
- 数据和证据：比例、金额、时间、研究、新闻。
- 隐喻对象：迷宫、齿轮、火、桥、放大镜、考拉、流水线。

#### Step 4：选择素材关系

每个镜头必须标注素材关系：

| 素材关系 | 使用场景 | 示例 |
|---|---|---|
| 直接对应 | 文案有明确对象 | 讲 AI 工具就放工具界面 |
| 情绪对应 | 文案讲感受 | 焦虑时放深夜电脑前 |
| 隐喻对应 | 文案抽象 | 讲系统困境放迷宫/齿轮 |
| 论据对应 | 文案讲事实 | 放新闻截图/数据图 |
| 节奏对应 | 文案有转折/金句 | 大字压屏/定格/音效 |

优先级：

```text
真实证据 > 屏幕录制 > 具体场景 > 信息图/MG > 情绪 B-roll > 抽象隐喻
```

#### Step 5：决定素材来源

每个镜头选择一种或多种：

- 自己录屏：AI 工具界面、搜索过程、工作流演示。
- 真实截图：新闻、报告、评论、数据、收益、流程。
- 素材库 B-roll：办公、通勤、城市、人群、学习、焦虑。
- AI 生成视频：抽象场景、隐喻画面、无法拍摄的镜头。
- MG/信息图：流程、分类、时间线、对比、数字。
- 局部真人：手势、桌面、背影、走路、打字，增强真人感。

#### Step 6：输出分镜表

每个镜头至少包含以下字段：

| 字段 | 必填 | 说明 |
|---|---|---|
| 镜头编号 | 是 | Shot 01 |
| 时间/时长 | 是 | 0-3s |
| 分镜目的 | 是 | 留人/解释/情绪/论据/转折/金句/CTA |
| 旁白 | 是 | 当前镜头对应文案 |
| 画面内容 | 是 | 观众应该看到什么 |
| 可视化关键词 | 是 | 从文案提取的视觉词 |
| 素材关系 | 是 | 直接/情绪/隐喻/论据/节奏 |
| AI 视频提示词 | 是 | 英文优先，适合生成 3-5s 片段 |
| 素材库搜索关键词 | 是 | 中文 + 英文 |
| 字幕关键词 | 是 | 需要高亮的词 |
| 字幕样式 | 是 | 大字压屏/底部字幕/关键词变色/标题卡 |
| 转场/动画 | 是 | 硬切/推近/闪白/模糊/卡片 |
| 音效/BGM | 是 | 铺底/重音/停顿/转折音效 |
| 剪映操作 | 是 | 新手具体执行步骤 |
| 备注/风险 | 否 | 版权、画面难度、替代方案 |

### 3.5 分镜输出示例

| 镜头 | 时长 | 目的 | 旁白 | 画面内容 | AI 视频提示词 | 素材搜索关键词 | 字幕关键词 |
|---|---:|---|---|---|---|---|---|
| 01 | 0-3s | 留人 | 普通人学 AI，最怕的不是工具太多，而是学完以后不知道怎么用。 | 深夜，一个人坐在电脑前，屏幕打开多个 AI 工具页面，表情焦虑。 | A tired solo worker sitting at a desk late at night, multiple AI tool interfaces glowing on the monitor, overwhelmed expression, cinematic lighting, realistic, vertical video. | 中文：深夜办公、AI工具、焦虑、电脑屏幕；英文：AI tools, late night laptop, overwhelmed worker, productivity anxiety | 不是工具太多 / 是不知道怎么用 |

补充字段：

```text
可视化关键词：普通人、AI工具、学完不会用、焦虑、电脑
素材关系：情绪对应 + 直接对应
字幕样式：大字压屏，关键词黄色高亮
转场/动画：开头轻推近，0.5 秒后硬切

剪映操作：导入深夜办公素材，添加 105% 到 112% 推近关键帧，底部字幕保留，大字标题居中偏上。
```

### 3.6 剪辑模块库

#### 开头钩子模块

- 反常识问题：为什么 X 明明这样，却会那样？
- 强对比痛点：别人能做 6 小时，你 8 分钟就崩。
- 热点争议：最近大家都在吵 X，但真正的问题是 Y。
- 真实结果：三个月/四天收益/真实困境。
- 利他承诺：看完你会知道怎么做第一步。
- 账号定位：我不是讲黑话的，我用普通人听得懂的话讲 AI。

#### 字幕模块

- 主字幕：白字 + 黑描边，底部两行内。
- 关键词高亮：黄/红/青色，一屏最多 1-2 个。
- 金句大字：单独成屏，停顿 0.5-1 秒。
- 标题卡：黑底/网格/模糊背景 + 大字。
- 数据卡：白底/黑字/大数字，适合可信度。

#### 画面模块

- 现实场景：办公室、地铁、深夜电脑、通勤、人群。
- AI 场景：工具界面、提示词、流程图、自动化。
- 论据场景：新闻截图、报告、评论、数据图。
- 情绪场景：背影、空房间、低头、焦虑表情。
- 隐喻场景：迷宫、齿轮、桥、火、流水线、考拉。
- 局部真人：手、桌面、麦克风、打字、翻笔记。

#### 转场/节奏模块

- 默认硬切。
- 概念切换用标题卡。
- 金句用定格/轻推近/低频音效。
- 数据用闪白或卡片弹入。
- 情绪段降低切换频率，解释段提高信息密度。

#### 音频模块

- 人声优先，BGM 只做铺底。
- AI 合成声音必须标注停顿、重音、语速、情绪。
- 转折句前后留 0.3-0.8 秒停顿。
- 金句前可短暂停顿，金句后不要马上切走。
- 不建议新手过度堆音效。

### 3.8 质量检查

输出前检查：

- 每个镜头是否有明确目的。
- 是否每个镜头都有旁白、画面、AI 提示词、素材搜索词、字幕关键词。
- 画面是否服务文案，而不是空泛装饰。
- 每 5-10 秒是否有视觉或信息刺激。
- 数据和新闻是否需要来源。
- AI 视频提示词是否具体到主体、动作、场景、镜头、风格。
- 是否提醒版权风险，尤其影视片段、新闻截图、他人视频素材。
- 是否适合剪映新手执行。

## 4. 三个 Skill 的边界

### 4.1 内容 Skill 不做什么

- 不直接输出最终分镜表，只输出视觉锚点。
- 不编造数据、案例、新闻。
- 不承诺收益、涨粉、变现结果。
- 不把他人内容照搬成用户原创。
- 不把所有资料塞进文案，必须筛选。

### 4.2 剪辑 Skill 不做什么

- 不负责重新研究选题深度，除非用户要求补资料。
- 不机械地一句话一个镜头。
- 不默认 AI 一键生成整条视频。
- 不鼓励无版权影视素材搬运。
- 不为了炫技牺牲理解。

### 4.3 协作顺序

推荐流程：

```text
1. 用户给话题
2. 内容 skill 做研究和脚本
3. 用户确认脚本方向
4. 剪辑 skill 拆分镜
5. 用户按分镜找素材/生成素材/剪映执行
6. 发布后复盘数据
7. 把可复用经验沉淀回 references
```
